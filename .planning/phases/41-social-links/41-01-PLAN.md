<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| social icon anchors → external URLs | Static mock URLs in source code; no user input crosses this boundary |
| browser tab → opened external URL | New tab created via target="_blank"; parent page accessible via window.opener without rel="noopener" |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-41-01 | Elevation of Privilege | All social `<a>` elements with target="_blank" | mitigate | Include rel="noopener noreferrer" on every anchor — prevents opened tab from accessing window.opener and redirecting parent; enforced in SocialNavLinks component per D-05 |
| T-41-02 | Information Disclosure | rel="noopener noreferrer" | mitigate | noreferrer suppresses HTTP Referer header so external site does not learn which store page the user was on |
| T-41-03 | Tampering | Mock URLs in src/lib/social-links.ts | accept | URLs are static code, not user-supplied; no input validation surface; operator replaces values when real URLs are available — low risk, no PII |
| T-41-SC | Tampering | npm/pip/cargo installs | accept | This phase installs zero packages — no slopcheck gate needed |
</threat_model>

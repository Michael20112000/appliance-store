<objective>
Remove `{ shouldValidate: true }` from the `onChange` handler in the callback phone input, so that the Zod validation error "Вкажіть номер телефону — лише цифри, від 10 до 15" never appears while the user is still typing a partial number.

Purpose: FAB-03 — the premature error message appears because `shouldValidate: true` triggers the `^\d{10,15}$` Zod regex on every keystroke, even at 1–9 digits. With `mode: "onSubmit"` (react-hook-form default in this component), removing that flag restores the expected UX: silent typing, error only after a failed submit.

Output: One-line change in `callback-request-form.tsx`; two new tests in `callback-request-form.test.tsx` proving the before/after behaviour.
</objective>

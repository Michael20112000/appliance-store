<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| GitHub Actions → Neon CI branch | CI job holds DB credentials in encrypted secrets |
| CI logs → public | Must not echo secret values |
| Developer laptop → localhost | Same Playwright contract as CI |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-01 | Information disclosure | ci.yml env | mitigate | Use `secrets.*` only; document in checklist — no prod `DATABASE_URL` |
| T-06-02 | Tampering | CI pipeline | mitigate | Run `npm run lint` + `npm test` before Playwright; full `e2e/` directory |
| T-06-03 | Elevation | Optional E2E skips | accept | Skipped tests are not failures; required specs must stay green (D-06-24) |
| T-06-SC | Tampering | npm ci in CI | mitigate | Lockfile-only install via `npm ci`; no ad-hoc package adds in workflow |

</threat_model>

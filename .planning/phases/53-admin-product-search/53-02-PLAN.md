| Boundary | Description |
|----------|-------------|

| URL q param → Prisma | q flows through Zod validator → buildAdminProductWhere → parameterised Prisma query |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-53-01 | Tampering | q param → Prisma query | mitigate | `listAdminProductsSchema` (`q: z.string().max(100).optional()`) validates at page entry; `buildAdminProductWhere` uses Prisma `contains` (parameterised — never raw interpolation); already implemented, no change needed |
| T-53-02 | Tampering | XSS via q reflected in input value | mitigate | React renders `value={value}` as escaped string attribute; no `dangerouslySetInnerHTML`; safe by construction |
| T-53-03 | Denial of Service | DB hammering from rapid keystrokes | mitigate | 300 ms client debounce reduces query rate ~10x; `q.length >= 2` guard in service skips OR clause for single-char queries; `max(100)` Zod cap prevents extremely long queries; all three controls already implemented |
| T-53-04 | Spoofing | CSRF on search | accept | Search uses GET (read-only); no state mutation; CSRF not applicable to GET requests |
| T-53-05 | Elevation of Privilege | Unauthenticated access to /admin/tovary | accept | Admin layout gate-keeps all /admin/* routes via better-auth session check; ProductSearchInput adds no new auth surface |
| T-53-SC | Tampering | npm/pip/cargo installs | accept | No new packages in this phase; Package Legitimacy Audit in RESEARCH.md confirms zero new dependencies |

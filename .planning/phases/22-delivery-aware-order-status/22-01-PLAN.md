In `status-transitions.ts`:

1. Import `DeliveryType` from `@/generated/prisma/client` (or `@/types/order` if that re-exports it).
2. Add `isStatusCompatibleWithDelivery(status: OrderStatus, deliveryType: DeliveryType): boolean`:
   - return `false` when `status === "OUT_FOR_DELIVERY"` and `deliveryType === "PICKUP"`
   - return `false` when `status === "READY_FOR_PICKUP"` and `deliveryType === "LVIV_DELIVERY"`
   - otherwise `true`
3. Add `getAllowedNextStatusesForDelivery(from: OrderStatus, deliveryType: DeliveryType): OrderStatus[]` — filter `getAllowedNextStatuses(from)` with `isStatusCompatibleWithDelivery`.
4. Add `assertTransitionAllowedForDelivery(from, to, deliveryType)` — call existing `assertTransitionAllowed(from, to)` then if `!isStatusCompatibleWithDelivery(to, deliveryType)` throw `new Error(INVALID_STATUS_TRANSITION)`.

Create `src/lib/order/status-transitions.test.ts` with `describe.each` matrix covering at minimum:

| deliveryType | from | to | expect |
|--------------|------|-----|--------|
| PICKUP | CONFIRMED | READY_FOR_PICKUP | allow |
| PICKUP | CONFIRMED | OUT_FOR_DELIVERY | reject |
| LVIV_DELIVERY | CONFIRMED | OUT_FOR_DELIVERY | allow |
| LVIV_DELIVERY | CONFIRMED | READY_FOR_PICKUP | reject |
| PICKUP | PENDING | CONFIRMED | allow |
| either | COMPLETED | * | reject (existing graph) |

Use `assertTransitionAllowedForDelivery` and `getAllowedNextStatusesForDelivery` in tests; verify forbidden targets are absent from returned arrays.

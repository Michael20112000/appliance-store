# Milestone v3.0: Chat & Engagement

**Status:** ✅ SHIPPED 2026-05-26
**Phases:** 46–49
**Total Plans:** 16

## Overview

Full real-time chat system built on Pusher + PostgreSQL. Guest users can chat without registering (localStorage token); authenticated users get conversation history, an in-widget history drawer, and file attachment support. Admin gains chat lifecycle control (close chat, real-time notification) and can send image attachments.

## Milestone Summary

**Key Decisions:**

- Guest chat via localStorage `chat_guest_token` (UUID) — simple, no backend session creation
- `claimGuestConversation` wrapped in `$transaction` — TOCTOU race between findFirst + updateMany
- `Conversation.userId @unique` removed — hard prerequisite, enables multiple conversations per user
- `router.refresh()` after claim — SSR re-hydrates `initialConversationId` without page reload
- Pusher guest auth via `guestToken` param on existing `/api/pusher/auth` route (not a new channel type)
- History drawer as in-widget view-state switch (`panelView: 'thread' | 'history'`) — no Sheet/modal
- PDF support dropped in Phase 49 — images only (jpg/png/webp); Cloudinary signed preset `chat-attachments`
- File validation: type + size (10 MB) checked client-side; server re-validates via `chat-attachments` preset

**Issues Resolved:**

- Unique constraint on `guestToken` blocked second guest session — removed constraint in migration
- `unarchiveConversation` must set `isActive: true` (not just `status: "OPEN"`) — two-field guard
- Cloudinary PDF delivery returned 401 via `private_download_url` — switched to signed CDN URL
- `ConversationList` import missing in HistoryDrawer — inlined buyer list

**Issues Deferred:**

- PDF attachment support (requires delivery URL strategy beyond Phase 49 scope)
- File attachments for guests (abuse vector without moderation)
- Typing indicators (CHAT-10), read receipts (CHAT-11)
- Push notifications (NOTF-01/02)

**Technical Debt Incurred:**

- REQUIREMENTS.md checkboxes not synced during execution — confirmed shipped via phase summaries at milestone close
- `prisma/seed.test.ts` 2 pre-existing failures carried forward (unrelated to chat)

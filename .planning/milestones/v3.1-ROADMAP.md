## Milestone Summary

**Key Decisions:**

- DrawerProvider wraps ChatProvider (not vice versa) ‚Äî ensures StorefrontFabs and PdpCartFab can both useDrawers()
- Only one drawer open at a time (mutual exclusion in openCart/openWishlist)
- CartNavButton is a separate client component from CartNavLink RSC ‚Äî clean RSC/client boundary
- CSS translate overlay pattern for history panel ‚Äî PanelBody stays mounted, real-time message state preserved
- useState-based isOpen for chat (no nuqs/URL) ‚Äî chat survives page navigation without URL pollution
- Base UI Drawer (swipeDirection=down) for mobile chat ‚Äî native gesture support
- prefillText state lives in PanelBody (not ChatProvider) ‚Äî local UI state, not shared context
- SuggestedMessages hidden when messages.length > 0 OR isLoading OR !canSend
- adminProductsUrl always includes page when explicitly provided ‚Äî clearer URL for debugging
- isMountedRef guard in ProductSearchInput ‚Äî prevents spurious router.replace on mount

**Issues Resolved:**

- /koszyk and /obrane page navigation replaced entirely by drawer pattern
- Chat unread count changed from boolean to numeric for badge display (boolean‚Üínumber migration)
- Chat history content-swap replaced by CSS translate overlay (non-destructive)
- Chat URL dependency removed (nuqs search-params.ts deleted)
- Admin product filter URL builder page=1 omission bug fixed

**Issues Deferred:**

- None within v3.1 scope

**Technical Debt Incurred:**

- REQUIREMENTS.md checkbox sync gap (same recurring process miss as v3.0 ‚Äî 5 requirements marked Pending despite being shipped)

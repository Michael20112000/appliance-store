<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser → GET /api/chat/guest | Unauthenticated endpoint; guestToken is the only credential |
| Browser → POST /api/chat/messages | Mixed: session OR guestToken path; guests identified by UUID token |
| Browser → POST /api/chat/pusher/auth | Pusher channels are private; guestToken verified against DB before authorizing |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-46-06 | Information Disclosure | GET /api/chat/guest — token enumeration | accept | UUID guestToken has 2^122 entropy; 404 returns no info about valid tokens; acceptable risk |
| T-46-07 | Elevation of Privilege | POST /api/chat/pusher/auth — wrong guestToken | mitigate | DB lookup: `conversation.guestToken !== guestToken` → 403; guest cannot subscribe to another guest's channel |
| T-46-08 | Denial of Service | POST /api/chat/messages guest path — spam | mitigate | `sendMessage` calls `enforceRateLimit(guestToken)` — 20 msgs/min enforced at service layer |
| T-46-09 | Spoofing | POST /api/chat/messages — auth bypass via guestToken | mitigate | Guest path only accessible when `!session?.user`; session always checked first; guestToken rejected if session present |
| T-46-SC | Tampering | npm/pip/cargo installs | accept | Phase 46 installs zero new packages |
</threat_model>

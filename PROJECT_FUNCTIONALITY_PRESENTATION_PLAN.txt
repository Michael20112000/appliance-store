ПЛАН ПРЕЗЕНТАЦІЇ: ПОВНИЙ ФУНКЦІОНАЛ ПРОЄКТУ APPLIANCE-STORE (Поточний етап)

====================================================================
1) СТОРІНКА МАГАЗИНУ (Storefront) — ЩО БАЧИТЬ КІНЦЕВИЙ ПОКУПЕЦЬ
====================================================================

1.1. Глобальний каркас сайту
- Липкий хедер: логотип, категорії, соцмережі, кнопки "Обране" і "Кошик", вхід/реєстрація або кабінет/вихід.
- Мобільне меню (burger + sheet) з категоріями та контактами.
- Футер з телефонами, email, адресою, картою Google Maps та формою "Замовити дзвінок".
- Глобальні тости повідомлень для зворотного зв'язку користувачу.
- Плавні переходи між сторінками.

1.2. Головна сторінка
- Hero-блок з УТП магазину.
- Блок категорій з кількістю доступних товарів.
- Блок "Як купити" (покроковий процес для покупця).
- SEO JSON-LD для LocalBusiness.

1.3. Каталог товарів
- Загальний каталог (/katalog).
- Категорійні сторінки (/katalog/[slug]).
- Фільтри: категорія, бренд, ціна (діапазон), стан товару.
- Сортування: нові/ціна зростання/ціна спадання.
- Пошук по товарах.
- Активні фільтри у вигляді "чіпсів" з можливістю швидкого скидання.
- URL-синхронізація фільтрів (посилання можна шарити в точному стані).
- Адаптивні фільтри: сайдбар на десктопі, sheet на мобілці.
- Картки товарів з фото, ціною, станом, брендом, кнопкою обраного.
- Ховер-стек фото на картці (каруселька при наведенні на десктопі).
- Коректні empty-state сценарії.

1.4. Сторінка товару (PDP)
- Повна галерея фото + прев'ю + збільшення у модалці.
- Детальна інформація: назва, бренд, ціна, стан, опис.
- Додавання/видалення з кошика.
- Додавання/видалення з обраного.
- CTA "Запитати про цей товар" (відкриває чат з контекстом конкретного товару).
- Блок "Схожі товари".
- Product JSON-LD для SEO.
- Невалідні/продані товари віддаються як 404.

1.5. Кошик
- Окрема сторінка кошика (/koszyk).
- Глобальний drawer кошика (відкривається з хедера/FAB).
- Перегляд товарів у кошику, видалення позицій, очищення всього кошика.
- Блок підсумку: загальна сума, кількість, перехід до оформлення.
- Порожній стан з переходом до каталогу.
- Працює як для гостя (localStorage), так і для авторизованого користувача (БД).
- Автозлиття гостьового кошика після логіну/реєстрації.

1.6. Обране (Wishlist)
- Окрема сторінка обраного (/obrane).
- Додавання/видалення через сердечко з картки і PDP.
- Глобальний drawer обраного.
- Лічильник обраного у хедері.
- Працює для гостя (localStorage) і для авторизованого (БД).
- Автозлиття гостьового обраного після логіну/реєстрації.
- Ліміт на кількість елементів.
- Відображення недоступних товарів в обраному з явним статусом.

1.7. Оформлення замовлення (Checkout)
- Сторінка оформлення (/zamovlennia).
- Форма: ім'я, телефон, спосіб доставки, адреса (коли потрібна), коментар.
- Валідація всіх полів.
- Підсумок замовлення перед відправкою.
- Підтвердження замовлення на окремій сторінці (/zamovlennia/pidtverdzhennia/[orderNumber]).
- Підтримка гостьового checkout і авторизованого checkout.
- Після оформлення очищається кошик.
- Логіка обробки доступу до підтвердження (юзер / гість по токену/кукі).

1.8. Авторизація та акаунт
- Сторінки входу та реєстрації.
- Better Auth (email/password).
- Callback URL логіка: повернення користувача у потрібну точку після логіну.
- Особистий кабінет (/kabinet):
  - Прев'ю обраного.
  - Історія замовлень з деталями.
  - Швидкий доступ до чату.

1.9. Реалтайм чат клієнт-магазин
- Глобальний віджет чату (floating panel/button).
- Працює для гостей і для авторизованих користувачів.
- Старт чату з контекстом товару (з PDP).
- Історія повідомлень.
- Підтримка вкладень зображень (для дозволених сценаріїв).
- Realtime оновлення через Pusher (нові повідомлення, статуси).
- Статус "чат завершено" + можливість почати новий.
- Список попередніх розмов користувача.

1.10. Floating action buttons
- FAB "Замовити дзвінок".
- FAB "Кошик" з бейджем кількості.
- FAB "Чат з магазином" з індикатором непрочитаних.
- Safe-area відступи для мобільних пристроїв.

1.11. Форма "Замовити дзвінок"
- Швидка форма по номеру телефону.
- Валідація телефону.
- Обробка і збереження запиту.
- Тости успіху/помилок.

====================================================================
2) АДМІН-ПАНЕЛЬ (Backoffice) — ФУНКЦІОНАЛ ДЛЯ КОМАНДИ МАГАЗИНУ
====================================================================

2.1. Доступ і захист
- Вхід в адмінку тільки для ролі admin.
- Двошаровий захист:
  - Proxy/middleware редіректить неавторизованих.
  - Серверні перевірки requireAdmin в адмін-сторінках і діях.

2.2. Dashboard (/admin)
- KPI-картки: нові замовлення, залишки, дзвінки, непрочитані чати.
- Короткий аналітичний прев'ю-блок.
- Останні замовлення.
- Сайдбар-бейджі з кількостями "що горить зараз".

2.3. Аналітика (/admin/analityka)
- Періоди 7/30/90 днів.
- Метрики: кількість замовлень, виручка, дзвінки.
- Графіки замовлень/виручки по днях.

2.4. Керування категоріями (/admin/kategorii)
- Список категорій з кількістю товарів.
- Створення нової категорії.
- Редагування категорії.
- Завантаження/оновлення зображення категорії (Cloudinary).
- Drag-and-drop reorder категорій.
- Захист від видалення категорії, якщо в ній є товари.

2.5. Керування товарами (/admin/tovary)
- Таблиця товарів з фільтрами, пошуком, сортуванням і пагінацією.
- Створення товару.
- Редагування товару.
- Керування галереєю товару (до 8 фото, Cloudinary).
- Відстеження використання товару в замовленнях.
- Захист від видалення товару, якщо він у кошику/активних замовленнях.

2.6. Керування замовленнями (/admin/zamovlennia)
- Список замовлень з фільтрами по статусу.
- Детальна сторінка замовлення.
- Зміна статусу замовлення.
- Статус-машина переходів з правилами по типу доставки.
- Резерв/повернення складських залишків при зміні статусів.

2.7. Керування чатами (/admin/chaty)
- Inbox чатів (активні/архів).
- Відповідь адміну в конкретний чат.
- Архівування/розархівування/видалення чату.
- Realtime синхронізація подій.

2.8. Керування callback-запитами (/admin/dzvinky)
- Список активних і архівних заявок.
- Зміна статусу (PENDING/CONSULTED).
- Нотатки менеджера по заявці.
- Архівація після обробки.

2.9. Налаштування магазину (/admin/nalashtuvannia)
- Телефони, email, адреси магазину.
- Редагування і збереження контактних даних.
- Використання цих даних у storefront (хедер/футер/schema).

3) BACKEND/API/SERVER ACTIONS — ЩО РОБИТЬ СИСТЕМА ПІД КАПОТОМ
====================================================================

3.1. База даних і домен
- Prisma + PostgreSQL (Neon).
- Основні сутності: User/Session/Role, Category, Product, ProductImage, Cart, Wishlist, Order/OrderItem, Conversation/Message, CallbackRequest, Store contacts.
- Індекси під пошук/фільтри/статуси/дату.

3.2. Бізнес-сервіси
- Catalog service: публічні товари, фільтри, бренди, діапазон цін, схожі товари.
- Cart service: CRUD кошика + merge guest->user.
- Wishlist service: CRUD + merge guest->user + ліміт.
- Order service: створення замовлень (user/guest), перевірка наявності, генерація номера.
- Chat service: створення діалогів, відправка/читання, unread, claim guest chat, lifecycle.
- Callback service: створення і адмін-обробка заявок.
- Admin services: dashboard counts, analytics, CRUD категорій/товарів/замовлень.

3.3. API маршрути
- /api/auth/[...all] — Better Auth.
- /api/upload/sign — адмінський signed upload для Cloudinary.
- /api/chat/* — conversation/message/new/claim/guest/pusher auth/upload sign.

3.4. Валідація
- Zod-схеми на всі ключові payload-и (auth, checkout, cart, wishlist, chat, callback, admin entities).
- Серверні екшени працюють по патерну: guard -> validate -> service -> revalidatePath -> result.

5) SEO, ЛОКАЛІЗАЦІЯ, A11Y
====================================================================

5.1. SEO
- Metadata по сторінках.
- robots.txt.
- Dynamic sitemap.
- JSON-LD: LocalBusiness + Product.
- Noindex для технічних/нерелевантних сторінок (адмінка, певні фільтровані стани).

5.2. Локалізація і форматування
- Мова інтерфейсу: українська.
- Валюта: UAH (відображення в ₴, зберігання в копійках як integer).
- Українські формати телефона/чисел/плюралізації.

5.3. Доступність
- Семантичні лейаути (header/main/footer/nav).
- aria-атрибути.
- focus-visible стани.
- Підтримка prefers-reduced-motion.
- Адаптивність на мобільних.

8) ГОТОВИЙ СЦЕНАРІЙ ПРЕЗЕНТАЦІЇ ДЛЯ ЗАМОВНИКА (короткий порядок)
====================================================================

Крок 1: Показати storefront UX:
- Головна -> каталог -> фільтри/сортування -> картка товару -> схожі товари.

Крок 2: Показати покупку:
- Додати в кошик/обране -> checkout -> confirmation -> кабінет/історія замовлень.

Крок 3: Показати комунікацію:
- Callback форма + realtime чат з контекстом товару.

Крок 4: Показати адмінку:
- Dashboard -> товари -> категорії -> замовлення -> чати -> callback -> налаштування магазину.

Крок 5: Показати "enterprise-ready" частину:
- Безпека доступів, валідації, SEO, аналітика, тести, CI/CD readiness.

====================================================================
9) ВАЖЛИВЕ УТОЧНЕННЯ ПО МЕЖАХ ПОТОЧНОГО ЕТАПУ
====================================================================

Реалізовано на поточному етапі:
- Повноцінний storefront + admin + checkout + chat + callbacks + аналітика + тестове покриття.

Відсутнє на поточному етапі (щоб не оверпродати на презентації):
- Онлайн-еквайринг платежів на сайті (оплата зараз при отриманні).
- Повноцінна багатомовність (інтерфейс зараз україномовний).
- Явні фонові cron/queue джоби як окремий модуль (core функції працюють без них).
